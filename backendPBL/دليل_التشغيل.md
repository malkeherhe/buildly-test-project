## 🔷 ما هو venv؟

**venv (Virtual Environment)** هي بيئة افتراضية معزولة لمشروع Python.

### لماذا نحتاجها؟
- ✅ عزل الحزم: كل مشروع له حزمه الخاصة
- ✅ تجنب التعارضات: منع تضارب إصدارات المكتبات
- ✅ سهولة النقل: يمكن مشاركة المشروع بسهولة
- ✅ التنظيم: إدارة أفضل للمشاريع

**نعم، يجب استخدام venv لكل مشروع Python!**

## 🚀 إعداد الباك إند

### الخطوة 1: فتح Terminal/PowerShell
افتح PowerShell أو Command Prompt في مجلد المشروع:
```
C:\Users\bassa\OneDrive\سطح المكتب\project\backendPBL
```

### الخطوة 2: تفعيل البيئة الافتراضية (venv)

لديك venv موجود بالفعل في المجلد الرئيسي. قم بتفعيله:

**في PowerShell:**
```powershell
.\venv\Scripts\Activate.ps1
```

**في Command Prompt:**
```cmd
venv\Scripts\activate.bat
```

**ملاحظة:** إذا ظهرت رسالة خطأ في PowerShell حول "execution policy"، قم بتشغيل:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### الخطوة 3: التحقق من التفعيل
بعد التفعيل، يجب أن ترى `(venv)` في بداية السطر:
```
(venv) PS C:\Users\bassa\...>
```

### الخطوة 4: الانتقال لمجلد المشروع
```powershell
cd projectBPL
```

### الخطوة 5: تثبيت الحزم (إذا لم تكن مثبتة)
```powershell
pip install -r requirements.txt
```

**الحزم المطلوبة:**
- Django==4.2.0
- djangorestframework==3.14.0
- djangorestframework-simplejwt==5.3.0
- django-cors-headers==4.2.0
- python-decouple==3.8

### الخطوة 6: تطبيق Migrations (إنشاء قاعدة البيانات)
```powershell
python manage.py migrate
```

## ▶️ تشغيل الباك إند

### الخطوة 1: التأكد من تفعيل venv
تأكد أن `(venv)` موجود في السطر

### الخطوة 2: تشغيل السيرفر
```powershell
python manage.py runserver
```

### الخطوة 3: النتيجة المتوقعة
يجب أن ترى رسالة مثل:
```
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.
```

**🎉 الباك إند يعمل الآن على:** `http://localhost:8000`

## ✅ التحقق من أن الباك إند يعمل

### الطريقة 1: فتح المتصفح
افتح المتصفح واذهب إلى:
```
http://localhost:8000
```

**النتيجة المتوقعة:** صفحة Django الافتراضية أو رسالة خطأ 404 (هذا طبيعي)

### الطريقة 2: اختبار API مباشرة
افتح المتصفح واذهب إلى:
```
http://localhost:8000/api/courses/
```

**النتيجة المتوقعة:**
- إذا لم تكن مسجل دخول: رسالة خطأ 401 (Unauthorized) - هذا يعني أن الباك إند يعمل!
- إذا كنت مسجل دخول: قائمة المسارات

### الطريقة 3: استخدام Postman أو curl

**في PowerShell (curl):**
```powershell
curl http://localhost:8000/api/courses/
```

**النتيجة المتوقعة:** JSON response أو رسالة خطأ (كلاهما يعني أن الباك إند يعمل)

### الطريقة 4: اختبار endpoint بسيط
افتح:
```
http://localhost:8000/admin/
```

**النتيجة المتوقعة:** صفحة تسجيل دخول Django Admin

## ⚠️ حل المشاكل الشائعة

### مشكلة 1: "python: command not found"
**الحل:** استخدم `python` بدلاً من `python3` في Windows

### مشكلة 2: "npm: command not found"
**الحل:** قم بتثبيت Node.js من https://nodejs.org

### مشكلة 3: "ModuleNotFoundError"
**الحل:** تأكد من تفعيل venv وتثبيت الحزم:
```powershell
pip install -r requirements.txt
```

### مشكلة 4: "Port 8000 already in use"
**الحل:** استخدم منفذ آخر:
```powershell
python manage.py runserver 8001
```

### مشكلة 5: "CORS error" في المتصفح
**الحل:** تأكد من أن `django-cors-headers` مثبت وأن الباك إند يعمل

### مشكلة 6: "Execution Policy" في PowerShell
**الحل:**
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## 🔍 التحقق النهائي

### ✅ الباك إند يعمل إذا:
- Terminal يظهر: `Starting development server at http://127.0.0.1:8000/`
- المتصفح على `http://localhost:8000` يعرض صفحة Django
- `http://localhost:8000/api/courses/` يعطي response (حتى لو كان خطأ 401)

### ✅ الفرونت إند يعمل إذا:
- Terminal يظهر: `Local: http://localhost:3000/`
- المتصفح على `http://localhost:3000` يعرض واجهة التطبيق
- يمكنك رؤية صفحة تسجيل الدخول

## 📞 نصائح إضافية

1. **احتفظ باثنين من Terminals مفتوحة:**
   - Terminal 1: للباك إند (يعمل `python manage.py runserver`)
   - Terminal 2: للفرونت إند (يعمل `npm run dev`)

2. **لا تغلق Terminals أثناء العمل:**
   - الباك إند والفرونت إند يجب أن يبقيا يعملان

3. **لإيقاف السيرفر:**
   - اضغط `CTRL + C` في Terminal

4. **لإيقاف venv:**
   ```powershell
   deactivate
   ```
